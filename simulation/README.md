# Furuta Pendulum Control вЂ” PID Controller

Simulation and Simulink model for balancing a Furuta (rotary inverted)
pendulum using a single-sensor **PID** controller.

## What it does

A motor spins a horizontal arm (angle `theta`) about a vertical axis.
A pendulum hangs off the end of the arm and can swing freely (angle
`alpha`, where `0` = balanced upright). The goal is to apply motor
torque (`tau`) so the pendulum stays upright, even when a disturbance
pulse hits it at `t = 5 s`.

## Requirements

- MATLAB R2020b or later
- Simulink (only needed for `furuta_pid_option_b_model_build.m`)
- Control System Toolbox (`pidtune`, `ss`, `pid`, `feedback`)

## File overview

| File | Purpose |
|---|---|
| `furuta_params.m` | Physical constants (masses, lengths, inertias, damping, gravity) and the state/input convention. **Always run first.** |
| `furuta_plant_matlabfunction.m` | Nonlinear equations of motion вЂ” the ground-truth plant function shared by all scripts. |
| `furuta_pid_design.m` | Linearizes the plant (single output: `alpha`), tunes a filtered PIDF via `pidtune`, checks closed-loop stability, saves `furuta_pid_model.mat`. |
| `furuta_simulate_pid.m` | Runs the PID controller on the full nonlinear plant (`ode45`) with a disturbance pulse and plots `theta`, `alpha`, `tau`. |
| `furuta_pid_option_b_model_build.m` | Programmatically builds the PID Simulink model (`furuta_pid_option_b_model.slx`). |
| `furuta_pid_model.mat` | Saved PID gains: `Kp`, `Ki`, `Kd`, `Tf`. |
| `furuta_pid_option_b_model.slx` | Pre-built Simulink model (regenerated by the build script). |

## Quick start

```matlab
furuta_params                      % Load physical constants
furuta_pid_design                  % Tune gains -> furuta_pid_model.mat
furuta_simulate_pid                % Nonlinear check: plots theta, alpha, tau
furuta_pid_option_b_model_build    % Optional: build/rebuild the Simulink model
```

Always run `furuta_simulate_pid` before the Simulink model вЂ” it is
faster to iterate on and does not depend on the Simulink build step.

## Using the Simulink model

After running `furuta_pid_option_b_model_build`:

1. The model opens automatically.
2. Press **Ctrl+D** (Update Diagram) once вЂ” finalises MATLAB Function block ports.
3. Press **Run**. `alpha` should settle near `0` and hold through the disturbance at `t = 5 s`.

The model sets an `InitFcn` that auto-loads `Kp`, `Ki`, `Kd`, `Tf`
from `furuta_pid_model.mat` every time it opens вЂ” no need to re-run
the design script each session.

## Sign convention note

Both the design script, the simulation script (`furuta_simulate_pid.m`), and the Simulink model consistently use standard **negative** feedback:
